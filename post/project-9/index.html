Building a map of Traffic Signs

September 14, 2018

An Overview of the Project(STAR)

  • Situation: Develop a software tool for recognizing road signs in a given area of a map. The tool would be used to create a custom dataset of road coordinates and images, and for implementing traffic sign recognition

  • Task: To gather a dense set of road coordinates and images of the entire city of Luxembourg, and to accurately identify road signs in the city.

  • Action:

    • Data collection: I used Google’s map API to gather road coordinates in a grid pattern, used mechanism to retain only one reference per road, and finally gathered a dense dataset of 450K road coordinates.
    • Image collection: I collected 4 images for each road coordinate, resulting in 1.8 million images.
    • Image duplicate elimination: I grouped images by size and used MD5 hashing to identify similar images, then compared them to eliminate duplicates.
    • Collection of Negative Samples and Generation of Positive Samples: Collected around 4000 images without road signs as negative samples as they were in abundance. Automated the process of creating a positive set by using the negative set and augmenting it with road signs.
    • Traffic sign recognition: Initially, I implemented the Sliding Window approach with back-propagation but it was slow. So, I chose the HAAR cascade classifier as it was stage-based and fast once a model was trained. On top of this classifier, I used SIFT to extract feature descriptors of each road sign in order to eliminate false positives.

  • Result: Delivered a software that recognized road signs for a given geographical area with an accuracy of 77%. Specifically, the software was able to identify road signs for the entire city of Luxembourg
